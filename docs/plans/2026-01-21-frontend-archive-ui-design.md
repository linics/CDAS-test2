# Frontend Archive UI Design

## 背景与目标
当前前端页面视觉风格分散，缺少统一的质感与区分度。目标是构建“学术档案馆”风格的全站视觉系统，同时允许不同页面有明显区分。

## 视觉方向
- 风格：理性、克制、文献感、档案夹质感。
- 字体：标题衬线 + 正文无衬线，形成“论文标题 + 正文”结构。
- 色彩：纸感米白背景、墨绿/铁蓝为主色、暖金点缀。
- 动效：轻微淡入与上浮，交互强调细线高亮与柔和背景变化。

## 页面区分策略
- 登录/注册：封面页 + 目录卡风格，衬线大标题，盖章式按钮。
- 作业/资料库/提交列表：档案条目布局，编号/标签 + 摘要。
- 评价/批改：评审表样式，维度条块 + 明确刻度，总评独立框。
- 个人/小组/班级：档案夹式标签页切换。

## 组件与样式基线
- 统一卡片：细边框、纸张质感、轻阴影。
- 标题/副标题：衬线 + 无衬线组合。
- 标签/徽标：小型圆角、淡金边。
- 输入/按钮：细线边框 + 轻浮起效果。

## 落地范围
- 全局视觉变量与背景纹理：`frontend/src/index.css`, `frontend/src/App.css`
- 页面差异化：各页面局部样式覆盖（登录/注册/评审表为重点）
- 不引入新 UI 库，仅用 React + CSS

## 风险与注意事项
- 避免过度装饰影响可读性。
- 动效克制，保证性能与可访问性。
